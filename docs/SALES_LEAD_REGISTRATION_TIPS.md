# Sales Lead Registration Tips

Short, practical tips for registering leads in the inquiry and partner forms so data is consistent and useful for GRC plan inference and follow-up.

---

## 1. **Use KSA cities list (no free text)**

- Always pick **City** from the dropdown (Riyadh, Jeddah, Dammam, etc.).
- If the prospect is in a smaller city, use the closest major city or **Other**.
- Avoid typing city names manually so filters and reports stay consistent.

---

## 2. **Address**

- **Address line**: Use a single line (e.g. building name, district, street). No need for full multi-line unless you paste from a formal invite.
- Keep it short and readable for delivery and CRM.

---

## 3. **Company size (experience scale)**

- Choose the **employee band** that best matches the prospect (1–10, 11–50, 51–200, etc.).
- This drives expected value and GRC plan tier; pick the band the prospect described or that fits their sector.

---

## 4. **Expected value (SAR)**

- Use the **Expected value (SAR)** dropdown; don’t leave blank if the prospect shared a range.
- Bands: &lt;50k, 50k–200k, 200k–500k, 500k–1m, 1m+.
- Aligns with pipeline reporting and prioritization.

---

## 5. **Expected decision / signature date**

- Set **Expected decision or signature date** when the prospect gives a timeline.
- Use for follow-up and pipeline forecasting; update in admin if the date changes.

---

## 6. **Conditions or requirements**

- Use **Conditions or Requirements** for: compliance needs, payment terms, NDA, specific frameworks (e.g. NCA, SAMA), or other deal conditions.
- Keeps context in one place for legal/compliance and sales.

---

## 7. **Consistency for KSA leads**

- **Country**: Prefer **Saudi Arabia** for KSA; then pick a **KSA city** from the list.
- **Service area / product line**: Select the closest KSA CR activity so regulatory inference stays accurate.

---

## 8. **Partner-submitted leads**

- Use the same **KSA cities** dropdown and **address line** when submitting on behalf of a prospect.
- Fill **Company**, **Contact**, **Phone**, **Notes**; add **Expected value** and **Expected decision date** in admin if you have them.

---

## 9. **Quick checklist before submit**

- [ ] City from list (or Other)
- [ ] Address line if known
- [ ] Company size band
- [ ] Expected value (SAR) if discussed
- [ ] Expected decision/signature date if known
- [ ] Conditions/requirements in the text area

---

*Leads with complete, structured data get better GRC plan suggestions and faster follow-up.*
